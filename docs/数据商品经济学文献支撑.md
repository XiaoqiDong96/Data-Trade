# 数据作为商品的经济学文献支撑

生成日期：2026-06-14  
用途：支撑 RapidAPI `Data` 类别实证研究的理论动机、边际贡献和模型设定

## 一、研究对象：通过 API 交易的数据商品

本文研究的是通过 API 被商品化和市场化交易的数据。RapidAPI 的 `Data` 类别并非普通软件服务集合，而是一个公开数据商品市场：卖家将社交媒体、企业名录、地理位置、商品价格、房地产、金融、文档抽取等数据源封装为 API，买家购买的不是数据所有权，而是持续、可计量、可限制、可续订的数据访问权。

> 数据通过 API 被商品化以后，平台如何把“不可触摸、非竞争、质量事前不可完全观察、价值依赖用途、具有时效性和合规风险”的数据，转化为可搜索、可比较、可定价、可计量和可交易的商品。

因此，论文的文献支撑应围绕数据商品的六个特殊性展开：

1. 非竞争性和可重复销售。
2. 质量、覆盖范围和时效性事前难以验证。
3. 数据价值依赖买方用途，存在强异质性。
4. 数据常由其他活动副产品产生，边际复制成本低但采集/清洗/维护成本高。
5. 数据可能包含个人、企业或平台行为信息，存在隐私、产权和外部性。
6. 数据交易需要访问控制、用量计量和合规治理，而不是简单所有权转移。

RapidAPI 的 `Data` 类别正好把这些特点变成可观测变量：价格计划、免费试用、调用额度、超额费、速率限制、评分、订阅数、延迟、成功率、文档长度、更新时间、审批、私有计划和卖家组合。

## 二、核心文献脉络

### 1. 数据的非竞争性与产权安排

**Jones and Tonetti (2020), “Nonrivalry and the Economics of Data,” American Economic Review.**  
链接：[AEA](https://www.aeaweb.org/articles?id=10.1257/aer.20191330)

这篇是数据经济学最核心的理论文献之一。作者把数据的关键特征定义为非竞争性：同一份数据可以被多个企业同时使用而不被耗尽。非竞争性带来规模报酬递增，也带来配置问题：如果企业害怕竞争者利用数据，可能会囤积数据，导致数据使用不足；如果消费者拥有数据产权，配置可能更接近社会最优。

对本文的支撑：

- RapidAPI 上的数据 API 是非竞争性商品：一个 API 可以卖给大量订阅者，卖给一个买家并不减少卖给另一个买家的可能。
- 这解释了为什么平台普遍采用订阅费、额度、超额费，而不是“一次性出售所有权”。
- 这也解释了为什么数据商品可以存在免费计划：免费试用的边际复制成本低，卖家用它扩大需求和筛选买方。

可用于正文的理论表述：

> 与实体商品不同，数据具有非竞争性。平台上的 API 交易并不是转移数据所有权，而是出售对持续更新数据流的受控访问权。因此，数据商品的价格更接近访问价格和使用价格，而不是传统商品的单位所有权价格。

对应 RapidAPI 变量：

- `plan_price`
- `plan_monthly_price`
- `limit_amount_num`
- `limit_overage_price_num`
- `subscriptions_count`
- `is_free_plan`
- `is_paid_plan`

### 2. 数据经济中的增长、反馈和免费服务

**Farboodi and Veldkamp (2021/2025), “A Model of the Data Economy.”**  
链接：[NBER Working Paper](https://www.nber.org/system/files/working_papers/w28427/w28427.pdf)，[CEPR 版本](https://cepr.org/publications/dp20085)

这篇文献把数据建模为经济活动生成的信息资产。交易和使用产生数据，数据又改进预测、生产和匹配，从而形成数据积累和经济增长。CEPR 页面明确提到，模型可以解释为什么很多 app 是“免费”的，以及传统 GDP 可能漏计数据价值。

对本文的支撑：

- RapidAPI 中大量 `FREEMIUM` 和 `FREE` API 不应被解释为“没有价值”，而应理解为数据经济中的获客、学习和使用反馈机制。
- 数据 API 的价值不只体现在订阅费，也体现在持续使用、调用量、反馈和买方业务价值。
- 这支持将 `subscriptions_count` 作为数据商品被采用和使用价值的代理。

对应 RapidAPI 发现：

- API 产品层 `FREEMIUM` 占约 `69.80%`。
- 公开可见 plan 中 `FREE` 计划占约 `25.09%`。
- 免费计划和低价计划可以被解释为数据商品的试用入口和需求发现机制。

### 3. 数据外部性与数据中介

**Bergemann, Bonatti and Gan (2022), “The Economics of Social Data,” RAND Journal of Economics.**  
链接：[RAND/Wiley](https://onlinelibrary.wiley.com/doi/abs/10.1111/1756-2171.12407)，[Yale 页面](https://economics.yale.edu/economics-social-data)，[arXiv](https://arxiv.org/abs/2004.03107)

这篇文献研究数据中介如何从消费者处获得信号，再将信息转售给产品市场中的企业。核心机制是“social data”：一个人的数据可以预测其他人的行为，因此个人数据具有外部性。数据中介可以利用这种外部性降低信息获取成本，并从信息转售中攫取价值。

对本文的支撑：

- RapidAPI 中很多数据商品不是简单的“个人数据文件”，而是 LinkedIn、企业名录、社交媒体、IP/地理位置、房地产、商品价格等可预测市场行为的数据。
- 数据卖家作为中介，把分散的数据源标准化为 API，再卖给下游开发者和企业。
- 平台的价值在于把数据中介的产品变成可发现、可计量、可订阅的市场商品。

对应 RapidAPI 变量：

- `owner_slugifiedName`
- `parent_org_slugifiedName`
- `billingitem_name`
- `plan_visibility`
- `requires_approval`
- `rating`
- `subscriptions_count`

### 4. 数据所有权、交易成本和市场失灵

**Duch-Brown, Martens and Mueller-Langer (2017), “The Economics of Ownership, Access and Trade in Digital Data.”**  
链接：[SSRN](https://papers.ssrn.com/article=2914144)，[JRC 页面](https://publications.jrc.ec.europa.eu/repository/handle/JRC104756)

这篇 JRC 工作论文系统讨论数字数据的所有权、访问和交易。其核心观点是，数据市场存在多种市场失灵：数据范围经济带来的外部性、数据所有者的战略行为、数据交换中的交易成本、产权和隐私的不清晰。

对本文的支撑：

- RapidAPI 的平台机制可以被理解为降低数据交易成本的制度安排。
- 搜索、分类、文档、价格菜单、订阅、调用计量和平台结算，都是为了让原本难以交易的数据变成可交易商品。
- private plan、approval、terms of service 和 legal document 字段体现了数据交易中的合约和合规成本。

对应 RapidAPI 变量：

- `plan_visibility`
- `is_private_plan`
- `requires_approval`
- `legalDocumentId`
- `legalAccountId`
- `termsOfService`
- `readme_len`
- `long_description_len`

### 5. 隐私、个人数据与信息外部性

**Acquisti, Taylor and Wagman (2016), “The Economics of Privacy,” Journal of Economic Literature.**  
链接：[AEA](https://www.aeaweb.org/articles?id=10.1257/jel.54.2.442)

这篇 JEL 综述系统总结隐私经济学，讨论个人信息披露的收益、成本、消费者行为偏差和隐私保护的经济后果。

**Choi, Jeon and Kim (2019), “Privacy and Personal Data Collection with Information Externalities,” Journal of Public Economics.**  
链接：[Journal of Public Economics / RePEc](https://ideas.repec.org/a/eee/pubeco/v173y2019icp113-124.html)，[TSE 页面](https://www.tse-fr.eu/publications/privacy-and-personal-data-collection-information-externalities)

这篇文章指出，即使消费者知道数据收集的后果，市场均衡中也可能出现过度收集个人数据，因为一个人的数据会泄露其他人的信息，产生信息外部性。

对本文的支撑：

- RapidAPI `Data` 类别中有大量 social、profile、email、IP、identity、location、lead、company enrichment 数据，这些产品与隐私和数据外部性直接相关。
- approval、private plan、terms of service 和公开/私有计划差异可以被解释为平台和卖家应对隐私/合规风险的机制。
- 隐私不必作为主模型的唯一机制，但应作为数据商品区别于普通数字商品的重要制度背景：交易对象可能包含第三方信息和外部性。

对应 RapidAPI 文本分类：

- `social_platform_data` 关键词覆盖约 `16.43%` API。
- `geo_identity_lookup` 关键词覆盖约 `17.32%` API。
- `firm_lead_business` 关键词覆盖约 `12.77%` API。

这些分类彼此重叠，但足以说明 RapidAPI 的 Data 类别有相当多产品涉及个人、企业或位置识别类信息。

### 6. 信息市场与数据中介的机制设计

**Bergemann and Bonatti (2019), “Markets for Information: An Introduction,” Annual Review of Economics.**  
链接：[Annual Reviews](https://www.annualreviews.org/content/journals/10.1146/annurev-economics-080315-015439)，[IDEAS/RePEc](https://ideas.repec.org/a/anr/reveco/v11y2019p85-107.html)

这篇综述将信息市场放在消费者、信息中介和企业的综合框架中，讨论信息如何被设计、销售和用于产品市场。它为将 RapidAPI 卖家理解为“数据/信息中介”提供理论基础。

对本文的支撑：

- RapidAPI 卖家不只是技术服务商，更是信息商品的设计者。
- 一个数据 API 的价值取决于信息结构：覆盖哪些对象、更新频率、端点设计、字段丰富度、调用稳定性、文档透明度。
- 多个 pricing plans 可以被理解为卖家针对不同买方用途设计不同版本的信息商品。

### 7. Arrow 信息悖论：为什么数据需要试用、声誉和平台治理

**Arrow (1962), “Economic Welfare and the Allocation of Resources for Invention.”**  
链接：[NBER chapter](https://www.nber.org/system/files/chapters/c2144/c2144.pdf)，[IDEAS/RePEc](https://ideas.repec.org/h/nbr/nberch/2144.html)

Arrow 的经典观点是，信息商品存在交易悖论：买家在购买前需要了解信息质量，但一旦完全了解信息，就已经获得了信息本身。这个悖论导致信息市场难以依靠普通商品交易机制运行。

对本文的支撑：

- 数据买家在订阅前无法完全知道数据是否准确、覆盖是否充分、更新是否及时、字段是否适合业务。
- RapidAPI 用免费计划、文档、评分、订阅数、成功率、延迟和服务水平来缓解 Arrow 信息悖论。
- 因此，声誉和质量信号不是普通平台装饰，而是数据商品交易得以发生的核心制度。

对应 RapidAPI 变量：

- `is_free_plan`
- `readme_len`
- `rating`
- `rating_votes`
- `subscriptions_count`
- `avg_success_rate`
- `avg_latency`
- `avg_service_level`

### 8. 信息商品版本化与菜单定价

**Varian (1997), “Versioning Information Goods.”**  
链接：[Berkeley PDF](https://people.ischool.berkeley.edu/~hal/Papers/version.pdf)

**Shapiro and Varian (1998), Information Rules.**  
链接：[书籍页面](https://www.amazon.com/Information-Rules-Strategic-Network-Economy/dp/087584863X)

Varian 的版本化理论指出，信息商品的生产固定成本高、复制边际成本低，因此适合通过版本化进行二级价格歧视。卖家可以通过质量、延迟、功能、限制等维度设计不同版本，让不同支付意愿的消费者自我选择。

对本文的支撑：

- RapidAPI 中大量 API 使用 `BASIC`、`PRO`、`ULTRA`、`MEGA` 四档计划，这正是信息商品版本化。
- 数据商品的版本差异不是传统物理质量，而是：调用额度、rate limit、更新频率、endpoint 范围、credits、超额费、审批权限。
- 这使 plan-level 和 plan-limit-level 面板成为本文的重要贡献。

对应 RapidAPI 发现：

- `BASIC`：`6892` 条。
- `PRO`：`5518` 条。
- `ULTRA`：`4959` 条。
- `MEGA`：`4217` 条。
- API 平均公开可见 plan 数约 `3.06`。

### 9. 信息商品捆绑定价

**Bakos and Brynjolfsson (1999), “Bundling Information Goods: Pricing, Profits, and Efficiency,” Management Science.**  
链接：[INFORMS](https://pubsonline.informs.org/doi/10.1287/mnsc.45.12.1613)，[IDEAS/RePEc](https://ideas.repec.org/a/inm/ormnsc/v45y1999i12p1613-1630.html)

这篇文献研究大量信息商品的捆绑定价，说明信息商品边际复制成本低、消费者估值异质时，捆绑可以提高利润并影响效率。

对本文的支撑：

- RapidAPI 的 plan 不是简单“一个请求一个价格”，而是把请求数、端点、credits、速率、超额费捆成套餐。
- 数据商品常被做成“包”：每月多少 requests、多少 credits、多少 rows、多少 searches。
- 因此，`billinglimits` 可以被理解为数据访问权的捆绑边界。

对应 RapidAPI 变量：

- `billingitem_name`
- `limit_amount_num`
- `limit_type`
- `limit_overage_price_num`
- `limits_n`
- `features_count`

### 10. 数据作为创新和市场力量投入品

**Beraja, Yang and Yuchtman (2023), “Data-intensive Innovation and the State: Evidence from AI Firms in China,” Review of Economic Studies.**  
链接：[Review of Economic Studies 页面](https://www.restud.com/data-intensive-innovation-and-the-state-evidence-from-ai-firms-in-china/)，[NBER/RePEc](https://ideas.repec.org/p/nbr/nberwo/27723.html)

该文用中国 AI 企业为例，说明政府数据的收集和提供会影响 AI 创新。它强调数据不仅是交易品，也是下游创新投入品。

**Begenau, Farboodi and Veldkamp (2018), “Big Data in Finance and the Growth of Large Firms,” Journal of Monetary Economics.**  
链接：[NBER](https://www.nber.org/papers/w24550)，[Princeton 页面](https://collaborate.princeton.edu/en/publications/big-data-in-finance-and-the-growth-of-large-firms/)

该文说明 big data 可以降低大企业资本成本，数据优势可能强化大企业增长。

对本文的支撑：

- RapidAPI 上的数据 API 的买家不是为“消费数据”而买数据，而是把数据作为输入品：做营销、风控、AI 训练、网页数据获取、市场监测、地址验证、金融分析等。
- 数据商品需求由下游用途决定，因此价格和订阅数会强烈依赖 API 所处的数据类型、覆盖范围和更新速度。
- 这支持将 `Data` 类别内部进一步分成 social、geo、finance、e-commerce、real estate、document/OCR 等细分市场。

## 四、RapidAPI 上“数据商品”的独特可观测机制

### 1. 数据的访问权而非所有权

数据的非竞争性意味着卖家不需要把数据所有权转让给单个买家。RapidAPI 的交易对象更像“持续访问权”：

```text
买方购买的是某个 API 在某个周期内的调用权、额度和超额访问权。
```

这与 Jones and Tonetti (2020) 的非竞争性逻辑一致，也与 Duch-Brown et al. (2017) 的 access/trade 讨论一致。

### 2. 数据质量不可完全事前验证

数据买家关心的是：

- 覆盖多少对象。
- 字段是否完整。
- 是否实时更新。
- 成功率是否高。
- 延迟是否低。
- 数据是否准确。
- 是否合规。

这些质量在购买前难以完全验证，因此平台需要免费计划、评分、订阅数、延迟、成功率、文档和服务水平来缓解信息悖论。

### 3. 数据价值依赖用途

同一个 LinkedIn profile API 对销售团队、招聘团队、风控团队、学术研究者的价值不同。因此卖家不能只设一个价格，而会设计多个版本：

- 小额度免费试用。
- BASIC 给测试者或小团队。
- PRO/ULTRA/MEGA 给高强度用户。
- CUSTOM/private plan 给企业客户。

这对应 Varian 的信息商品版本化。

### 4. 数据有时效性

数据商品不同于普通软件功能。许多数据的价值随时间衰减，例如社交资料、商品价格、房地产挂牌、招聘信息、天气、金融市场数据。

文本统计显示：

- `real-time` 出现于约 `11.00%` API。
- `realtime` 出现于约 `5.25%` API。
- `live` 出现于约 `5.76%` API。
- `fresh` 出现于约 `3.77%` API。
- `latest` 出现于约 `1.16%` API。

这说明实时性和新鲜度是 Data API 的显性卖点。模型中可以使用：

- `updatedAt`
- 距样本日期的最近更新时间
- 文本中是否包含 `real-time` / `live` / `fresh`
- `avg_latency`

### 5. 数据以用量计费，因为边际价值和边际成本都与调用量相关

Data API 的主要 billing item 是 Requests：

- 全部额度规则中 `Requests` 或 `requests` 占绝大多数。
- 公开可见计划里，`Requests` 对应 `20288` 条 limit，覆盖 `6532` 个 API。

这说明平台把数据商品转化为“可计量请求”。买家不是买一份静态文件，而是买可重复查询、可编程调用的数据访问。

## 五、可以写进论文的边际贡献

现有经济学文献已经很好地解释了数据的非竞争性、隐私外部性、产权安排、信息中介和信息商品定价。但是这些文献大多停留在理论层面，或者研究大型平台内部的数据优势，很少直接观察到公开数据商品市场中的微观价格菜单。

本文的边际贡献可以写成：

> 本文提供了一个公开数据商品市场的近全量微观证据。不同于既有文献主要从数据非竞争性、隐私外部性或平台数据优势出发，本文直接观察数据商品如何在市场中被打包、定价和交易：卖家将数据访问权设计为多档订阅计划，并通过调用额度、超额费、速率限制、免费试用、审批和声誉信号来缓解数据商品的质量不确定性、用途异质性和合规风险。本文因此把“数据作为经济商品”的理论特征映射到可观测的价格-质量-用量菜单。

更短的版本：

> We study data as a traded commodity, not merely as an input owned by platforms. Using near-universe data from RapidAPI's Data category, we observe how data sellers transform nonrival, experience-like, time-sensitive, and compliance-sensitive data into standardized access contracts with prices, quotas, overage fees, and reputation signals.

## 六、建议的文献综述结构

论文的文献综述可以分成四段。

第一段：数据作为特殊经济商品。  
引用 Jones and Tonetti (2020)、Farboodi and Veldkamp (2021/2025)、Duch-Brown et al. (2017)。强调非竞争性、产权、访问交易和数据价值。

第二段：数据外部性、隐私和数据中介。  
引用 Bergemann, Bonatti and Gan (2022)、Acquisti et al. (2016)、Choi et al. (2019)。强调个人/社交/企业数据的外部性和合规风险。

第三段：信息市场、信息悖论和版本化定价。  
引用 Arrow (1962)、Bergemann and Bonatti (2019)、Varian (1997)、Bakos and Brynjolfsson (1999)。强调试用、声誉、菜单、额度和捆绑。

第四段：实证贡献。  
说明现有文献缺少公开数据商品市场的微观价格和合约证据。RapidAPI 提供了 API-product、plan、limit 三层数据，使本文可以研究数据商品如何被平台标准化、如何定价、如何通过声誉和用量机制被交易。

## 七、可直接用于论文的段落

数据不同于普通数字服务。首先，数据具有非竞争性，同一份数据可以被多个买方同时使用而不被耗尽，这使数据交易更接近访问权销售而非所有权转移。其次，数据质量具有经验品属性，买方在购买前难以完全验证数据的覆盖范围、准确性、新鲜度和适配性，从而产生 Arrow 式信息悖论。再次，数据价值高度依赖下游用途，同一数据集对不同买家的价值差异很大，因此卖家有动机通过多档套餐、调用额度和超额费用进行二级价格歧视。最后，许多数据商品涉及个人、企业或平台行为信息，存在隐私、产权和外部性问题。

RapidAPI 的 Data 类别为研究这些机制提供了一个可观测市场。平台将数据商品标准化为 API 访问合约：一个 API 可以包含多个价格计划，每个计划规定月费、调用额度、速率限制和超额费用。平台同时展示订阅数、评分、成功率、延迟、服务水平和文档等声誉与质量信号。这些机制共同解决数据商品交易中的质量不确定性、用途异质性和交易成本问题。

与现有数据经济学文献相比，本文的贡献在于提供公开数据商品市场的近全量微观证据。本文不仅观察 API 产品本身，也观察其价格菜单和用量合约，从而能够把数据非竞争性、信息悖论、版本化定价和隐私/合规风险映射到真实市场中的价格、额度、超额费、免费计划和声誉机制。
